Consola de scripts para CONTPAQi Comercial · C# · Python · SQL

2.24.0 actual

2026-07-11 — ctx.show_html(): ventana HTML/WebView2 embebida (Python)
Agregado
  • ctx.show_html(html, title, width, height, modal) (Python): muestra una ventana con HTML/CSS/JS renderizado por WebView2, embebida dentro de CONTPAQi — sin escribir archivos ni depender del navegador externo. Primer caso real: el dashboard de flota de GGV_DE_MEXICO (miles de movimientos, filtros, exportación a Excel, todo dentro de una sola ventana).
  • Los datos que no quepan comprimidos bajo ~2MB (límite real de WebView2) se pueden comprimir con gzip+base64 y descomprimir en el navegador con la API nativa DecompressionStream — sin necesidad de ninguna librería extra.
  • PLANTILLA_DISENADOR_FORMULARIOS_PYTHON.py: diseñador visual de formularios no-code construido sobre ctx.show_html.

2.23.0

2026-07-03 — Ejemplo Premium: Recepción de Compra y Factura de Compra
Agregado
  • Recepción de Compra (módulo 184): transforma una o varias Órdenes de Compra del mismo proveedor en una Recepción, con partidas consolidadas por producto y captura de lote (+ caducidad) o número de serie según lo requiera el producto.
  • Factura de Compra (módulo 152): a partir de Órdenes de Compra seleccionadas en el grid de Comercial, con impuesto editable por partida y desglose de importes por línea. No afecta inventario; genera la póliza contable automáticamente.

2.21.12

2026-07-03 — Fin de la saga: sin transacción SQL + Unicode al guardar/cargar scripts
Corregido
  • NuevoDocumento/AgregarArticulo ya no envuelven el INSERT en un BEGIN TRANSACTION/COMMIT explícito — eso era lo que seguía causando "la operación no está permitida si el objeto está cerrado" al crear documentos desde un botón. Confirmado creando una Orden de Compra real.
  • Guardar y cargar un script desde la Consola podía dañar acentos/emoji (se veían como "?" o "♦") — ahora ambas rutas prefieren una conexión directa a la base con Unicode garantizado, con respaldo automático si no está disponible.

2.21.9

2026-07-02 — Saga completa: "busy" de XEngine + "objeto cerrado" en botones Python
Corregido
  • El botón "Ejecutar" de la Consola corría Python de forma síncrona (bloqueaba Comercial mientras la ventana seguía abierta) — ahora corre en segundo plano igual que el botón del ribbon, y hay una guardia contra ejecuciones encimadas por clics repetidos.
  • NuevoDocumento podía fallar con "la operación no está permitida si el objeto está cerrado" solo como botón de ribbon (nunca desde la Consola): la conexión a la base de datos ligada al grid activo se puede cerrar si el grid se refresca mientras una ventana interactiva sigue abierta varios minutos. Ahora se prefiere una conexión de propósito general y se revalida/renueva sola si deja de servir.
  • Los mensajes de error de NuevoDocumento/AgregarArticulo ahora incluyen el detalle real del error SQL/COM, no solo un mensaje genérico.
  • Un botón guardado desde Plantillas podía guardar acentos/emoji como "?" — corregido (lectura de archivos con codificación UTF-8 explícita).

2.18.0

2026-06-29 — 4 anclas + campos universales + partida nativa completa
Agregado
  • 4 anclas automáticas: NuevoDocumento ahora crea las 4 filas ancla 1:1 que todo documento nativo requiere: docDocumentExt, docDocumentExtra, docDocumentCFD y docDocumentPaymentAgenda. Antes solo creaba docDocumentExt.
  • Campos universales de docDocument: NuevoDocumento ahora llena MustBeSynchronized=1, ExportID=1, DateCost, DateDocDelivery, DateFrom, DateTo, DateLastPayment — igual que el nativo. Ya no se necesita UPDATE posterior para estos campos.
  • Partida como el nativo: AgregarArticulo ahora llena ApplyGlobalDiscount=1, DeductiblePerc=1, IsBusinessOperation=1, MustBeDelivered=1, DateItem, CoefUnit=1, ClaveUnidad y ObjetoImpuesto del producto.
  • API ctx.erp.LastError: expone el último error de late-binding COM. Revisar después de operaciones críticas (AffectStockNEW, Save, CancelDocument, Delete).
  • Fix encoding Python: el host ahora configura UTF-8 explícito en stdin/stdout/stderr del proceso Python. Corrige el bug Ñ→Ã.
  • MANUAL.md reescrito: documentación completa de ctx.erp (84 métodos, fichas por función, recetas por tipo de documento, advertencias).
Corregido
  • Encoding UTF-8 en comunicación host↔Python (datos con acentos/Ñ ya no se corrompen).
  • Com.LastError ahora captura y expone el mensaje de excepción de cada llamada COM fallida (antes se tragaba en silencio).

2.17.0

2026-06-28 — Documentos más fieles al nativo (docDocumentExt + costo de entrada)
Agregado
  • Documentos más fieles al nativo: al crear un documento con ctx.erp.NuevoDocumento(...) ahora se genera también la fila ancla en docDocumentExt (igual que CONTPAQi nativo), necesaria para los campos extra del documento.
